# Shell Tool: Execute Terminal Commands

PocketPaw includes a shell tool that allows your AI agent to execute terminal commands on your machine. Every command passes through **Guardian AI** before execution, ensuring unsafe or destructive commands are blocked automatically.

## How It Works

The shell tool provides:

1. **Command execution** - Run any bash/shell command on your system
2. **Output capture** - Returns stdout and stderr to the agent
3. **Working directory** - Commands run within the configured file jail
4. **Guardian AI check** - Every command is reviewed by a secondary LLM before running

### Why Guardian AI?

Before any shell command runs, a secondary LLM analyzes it for dangerous patterns:

- **Blocks destructive commands** - e.g. `rm -rf /`, `format`, `dd`
- **Audit logging** - Every command is logged to an append-only audit file
- **File jail** - Agent is restricted to allowed directories only
- **Panic button** - Instantly halt all shell activity from Telegram or dashboard

## Usage

```
User: Check how much disk space is left

Agent: [uses shell tool]
  → run command: df -h
  → returns: Filesystem usage table
  → "You have 45GB free on your main drive."

User: Find all Python files modified today

Agent: [uses shell tool]
  → run command: find . -name "*.py" -newer $(date +%Y-%m-%d)
  → returns: list of matching files
```

## Configuration

The shell tool is enabled by default. You can restrict it via tool policy:

```json
{
  "tool_policy": {
    "shell": "confirm"
  }
}
```

Available policy values:

| Value | Behavior |
| --- | --- |
| `allow` | Execute without confirmation (default) |
| `confirm` | Ask user before every command |
| `deny` | Disable shell tool entirely |

## Security

The shell tool is protected by multiple layers:

```
Command Request
      ↓
Guardian AI Review  ← blocks dangerous patterns
      ↓
File Jail Check     ← restricts working directory
      ↓
Audit Log Entry     ← append-only record
      ↓
Execution
```

To review the audit log:

```bash
cat ~/.pocketpaw/audit.log
```

To run a security audit:

```bash
pocketpaw --security-audit
```

## Policy Group

The shell tool belongs to `group:shell`. It is included in all default profiles but can be disabled for restricted environments.

## Environment Variables

```bash
POCKETPAW_SHELL_TIMEOUT=30        # Max seconds per command (default: 30)
POCKETPAW_FILE_JAIL=~/workspace   # Restrict agent to this directory
```
